[5] 브랜치 생성 , github에는 브랜치 라는 기능을 제공한다
	1. 목적 : 1. 협업시 각 브랜치는 각 사람들의 코드 구분/분류 , 2. 버전관리 : 테스트용코드용, 배포용코드, 임시코드 등등
	2. 정의 : 코드를 저장하는 하나의 저장소에 각 브랜치마다 서로 다른 저장 상태/버전 관리할수 있도록 지원하는 기능
		- 기본값 브랜치는 master/main
		- master/main 브랜치를 이용한 여러사람이 동일한 저장소를 사용하면 상태/버전 관리 힘들다.(  충돌  )

	3.협업에서 브랜치 활용
		1. master 브랜치는 취합(여러사람의코드를 합친결과-최신코드) 코드
		2. 각 팀원들 마다 각 브랜치를 만들어서 작업 하기
			팀원 : '유재석'(주인장) , '강호동' , '신동엽'
			브랜치 수 : master브랜치 , '유재석'브랜치 , '강호동'브랜치 , '신동엽'브랜치

		3 각 팀원들은 브랜치 만들기
			1. 주인장 : 프로젝트내 상단 [master/main] 클릭 -> new branch -> 브랜치명:한글본인이름(나중에영문)
			2. 초대받은사람 :
				1. 인텔리제이 최초 화면 에서 [ get from VCS  ]
				2.  왼쪽메뉴 [GITHUB] --> 리포지토리 명 검색 후 원하는 리포지토리 선택 -> [clone]
				3. 프로젝트내 상단 [master/main] 클릭 -> new branch -> 브랜치명:한글본인이름(나중에영문)
            * 브랜치 생성하고 최초 push 하면 코드와 브랜치 상태도 같이 push가 된다. ( 브랜치 생성후 최초 push 1번 한다 )

[6] 각 브랜치들이 코드를 작성하고  [master/main] 가 각 브랜치들을 취합(코드를 합치는)하는 작업
